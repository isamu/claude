# Claude Code 運用ガイド

## 品質レベルに応じた使い分け

### プロダクションコード（きちんと管理するソフトウェア）

- lint / unit test / formatter は必須
- CI でもしっかりテストを回す
- 指示は粒度を細かくし、出力に対してコードレビューを行う
  - DRY でないコード、大きな関数、CLAUDE.md のルール無視が起きやすいため、レビューは重要
- AI レビュー等の GitHub ツール（CodeRabbit など）を導入し、そのフィードバックを Claude に読み込ませて対応させる
- PR はレビュー後にマージ

### プロトタイプ・使い捨てツール

- 粒度の粗い指示でも OK
- 動作確認 + 簡単な PR で十分

## 活用範囲

プログラミングに限らず、ドキュメント作成やツールの定期実行にも活用できる。LLM を AI OS として捉え、あらゆるタスクの実行レイヤーとして使う。

## 自動化の方針

### Skill で定型化

- 定型タスクやリポジトリ固有のワークフローは skill として作成する

### プログラム化 vs Skill

- プログラム化できるもの → コードとして書き、unit test / generator を作る
- プログラム化が難しいもの → skill で code + LLM を組み合わせて対応する
